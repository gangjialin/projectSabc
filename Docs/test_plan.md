# 教师课程教学质量评分系统 — 测试方案（test_plan.md）

**版本**：v1.0
**日期**：2026-06-05
**依据文档**：
1. `Docs/requirements.md` v3.0（需求）
2. `Docs/design.md` v2.0（设计，§9 测试策略）
3. `Docs/tasks.md` v1.0（任务拆解，M1–M10）
4. `Docs/coding_standard.md`（§6 测试规则）

**核心原则**：本系统是合规性系统，**计算正确性 = 教师的实际利益**。测试重心放在
①5 维度计分与维度否决 ②等级比例划定 ③学生匿名 ④免计入排除 ⑤现场并发，五条主线绝不允许回归。

| # | 目标 | 关键关注 |

| 6 | 验证可用性与无障碍 | 移动端打分、表单可达性、加载/空/错状态 |

## 四、单元测试（重点：计算引擎）

> 对应任务 T-401~T-411。计算引擎是最高风险区，逐函数 + 边界 + 属性测试。

### 4.1 单次评分维度计算（DimensionScoreService）

| 用例 | 输入 | 期望 |
|---|---|---|
| U-CALC-01 | 某维度全选"完全符合"(5) | 维度得分=维度满分，得分率=1.0 |
| U-CALC-02 | 某维度全选"基本符合"(3) | 得分率=0.60 |
| U-CALC-03 | 单题得分 | = 题目分值 × likert/5 |
| U-CALC-04 | 5 维度满分求和 | totalScore=100（20+25+20+20+15） |
| U-CALC-05 | 题目分值之和校验 | 同维度题目分值和 ≠ 维度满分 → 抛错（数据完整性） |
| U-CALC-06 | 浮点累计 | 多题累加无精度漂移（统一保留位/四舍五入策略一致） |

### 4.2 跨评分人聚合 + 去极值

| 用例 | 场景 | 期望 |
|---|---|---|
| U-AGG-01 | 同行 ≥3 人 | 去掉一个最高一个最低后取均值 |
| U-AGG-02 | 同行 <3 人 | 取算术平均（不去极值） |
| U-AGG-03 | 同行恰好 3 人 | 去极值后剩 1 人即该人分 |
| U-AGG-04 | 上级听课多次 | 听课均分正确；材料均分正确 |
| U-AGG-05 | 极值并列 | 多个最高/最低时只各去其一（明确边界） |

### 4.3 三维加权与综合分（ScoreService）

| 用例 | 期望 |
|---|---|
| U-SCORE-01 | 上级=听课×60%+材料×40% |
| U-SCORE-02 | 学生=问卷×80%+访谈归一化(÷20×100)×20% |
| U-SCORE-03 | 综合分=上级×40%+同行×30%+学生×30% |
| U-SCORE-04 | 权重从 SystemConfig 读取（改配置后计算随之变化） |

### 4.4 维度否决判定（VetoJudgeService）⭐最高优先级

| 用例 | 场景 | 期望 |
|---|---|---|
| U-VETO-01 | 维度 n 加权得分率=0.69 | 触发否决，gradeRestriction=NO_A_OR_ABOVE |
| U-VETO-02 | 维度 n 加权得分率=0.70 | 恰好阈值，**不触发**（边界，<0.7 才触发） |
| U-VETO-03 | 多维度同时<0.7 | vetoDimensions 记录全部触发维度 |
| U-VETO-04 | 加权公式 | 维度n=上级n×0.4+同行n×0.3+学生n×0.3 |
| U-VETO-05 | 自动标记 isAutoGenerated=true | 系统生成，不可手工新增/篡改 |
| U-VETO-06 | 否决与 FORCE_D 共存 | FORCE_D 优先，最终 D |

### 4.5 等级划定（GradeJudgeService）⭐最高优先级

| 用例 | 场景 | 期望 |
|---|---|---|
| U-GRADE-01 | 比例划定 | S≤10%、S+A≈40%、B≈30%、C≈30%、D 不限 |
| U-GRADE-02 | 维度否决教师 | 即使综合分排名靠前，也不得评 A 及以上 |
| U-GRADE-03 | 中层干部 | 不占 S/A 配额，仅 S/A/D 三级 |
| U-GRADE-04 | 已认定 S | 从 B/C/D 自动划定中排除 |
| U-GRADE-05 | 学生分<90 | 不得 S |
| U-GRADE-06 | 责任事故 | 不得 B 及以上 |
| U-GRADE-07 | 教学差错 | 不得 A 及以上 |
| U-GRADE-08 | 边界并列分 | 按规则标记待人工商议，不静默处理 |

### 4.6 数据完整性检查

| 用例 | 不足条件 | 期望 |
|---|---|---|
| U-INTEG-01 | 听课<2 次 | 标记"数据不完整" |
| U-INTEG-02 | 学生问卷<10 份 | 红色警告 |
| U-INTEG-03 | 访谈委员<2 | 标记警告 |

## 五、API 集成测试

> 对应 design §5.2 接口清单。每个接口测：正常路径 + 校验失败 + 越权 + 边界。

| 接口 | 场景 | 期望结果 |
|---|---|---|
| `POST /auth/login` | 正确/错误密码、首登 | 返回 JWT；首登标记需改密 |
| `POST /import/teachers` | 合法/缺字段 Excel | 预览校验报错；确认后事务导入 |
| `POST /tasks/assign` | 委员评自己课程 | 拒绝（校验规则） |
| `POST /tasks/assign` | 同教师听课已 2 次再分 | 拒绝（≤2 次/年） |
| `POST /evaluation/submit` | 重复提交已锁定 | 拒绝（提交后锁定） |
| `POST /score/recalculate` | 触发计算 | BullMQ 入队、进度可查 |
| `GET /score/veto-list` | 查询否决名单 | 返回触发否决教师 |
| `GET /dimensions/teacher/:id` | 查 5 维度明细 | 三维各维度率 + 加权率 + 否决标记 |
| `POST /exemption` | 理由<30 字 / 超 5 条/学期 | 拒绝（防滥用 §7.4） |
| `POST /exemption/:id/dept-review` | 越级审核（跳过系部直接学院） | 拒绝（顺序约束） |
| `POST /exemption/:id/*-review` | 任一级驳回 | finalStatus=REJECTED，流程终止 |
| `POST /tracking` | 非 C 级建跟踪单 | 仅 C 级允许/自动 |
| `POST /appeal/college` | 超 3 工作日 | 拒绝/提示超期 |
| 全部敏感接口 | 无 token / 错角色 | 401 / 403（**后端鉴权**，不信任前端） |

## 七、实时与并发测试

| 用例 | 场景 | 目标 |
|---|---|---|
| RT-01 并发提交 | 100+ 同行同时提交说课打分 | 零丢分，全部入库 |
| RT-02 广播延迟 | 提交后大屏更新 | ≤3 秒 |

## 八、安全测试

| 领域 | 检查项 | 期望 |
|---|---|---|
| 认证 | JWT 过期/伪造/无 token | 拒绝访问 |
| 授权 | 越权访问他人数据、错误角色调用接口 | 403（后端校验，不信前端） |
| **学生匿名** | 教师端能否反查单个学生评分 | **不可**，只见聚合；审计表与公开记录分离 |
| **评分防篡改** | 已锁定提交被改 | 拒绝；解锁需 2 人审核(管理员+院长)并留痕 |
| **维度否决防篡改** | 手工新增/修改 DIM_VETO 标记 | 拒绝（系统自动生成只读） |
| 文件上传 | 非法类型/超大/恶意文件名 | 校验消毒后入 MinIO |
| 免计入防滥用 | 超额/短理由/利益相关方审核 | 拒绝 |
| 输入校验 | 边界注入/超长/类型错 | 边界统一校验 |
| 敏感数据 | 日志/错误响应 | 不泄露密码/内部错误栈 |
| 审计 | 解锁/改等级/建标记 | AuditLog 完整记录 |

## 十一、回归测试基线（每次发版必跑）

| 功能 | 回归风险 | 必跑用例 |

| 实时打分 | 并发/重连回归 | RT-* |

## 十二、测试数据策略

- **种子数据**：构造覆盖各等级、各前置限定、维度否决、中层干部、免计入的样例教师集。
- **黄金用例**：手工核算一份"标准答卷"的预期综合分与等级，作为计算引擎的回归断言基准（含 Excel 手算对照表）。
- **隔离**：每个用例独立准备/清理；CI 用一次性容器数据库。
- **隐私**：测试数据全部为虚构，不使用任何真实师生信息。

---

## 十三、验收标准（Definition of Done）

1. 计算相关单元测试覆盖率 ≥95%，整体 ≥80%。
2. 五条主线（计算/否决/匿名/免计入/并发）回归用例全绿。
3. 安全红线项 100% 通过。
4. 性能指标全部达标。
5. 关键 E2E 流程 100% 通过。
6. 无严重无障碍违规。
7. 黄金用例手算对照一致（误差为 0，或在约定的四舍五入规则内）。
