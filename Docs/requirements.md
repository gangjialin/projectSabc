# 教师课程教学质量评分系统 — 需求文档

**版本**：v3.0（依据学校《评价办法》V7.0 重大修订）
**日期**：2026-05-22
**依据文件**：
1. 《大连东软信息学院教师课程教学质量评价办法》V7.0（学校上位法）
2. 《数字艺术与设计学院教师课程教学质量评价实施方案》v2.0（学院实施方案）

**修订摘要（v2.0 → v3.0）**：
1. 评分体系全面对齐学校 5 维度框架（目标与思政20 + 内容与方式25 + 考核与评价20 + 效果与达成20 + 反思与改进15 = 100分）
2. 新增"任一维度<70% 不得 A 及以上"否决规则
3. 等级名称补全：S（示范级）/ A（发展级II）/ B（发展级I）/ C（关注级）/ D（不合格）
4. 课程覆盖比例调整：第1年70%、第2年100%、第3年全覆盖
5. 新增学生评价记录免计入申请流程
6. 新增持续改进跟踪单管理流程
7. 新增中层管理职务教师配合评价机制
8. 新增分课程类型评价（理论/实践/项目/毕设）
9. 申诉期限调整为3个工作日

## 一、项目背景与目标

### 1.1 背景

学院依据学校《评价办法》V7.0 和本院 v2.0 实施方案，实行"上级评价—同行评价—学生评价"三维评价模式，并严格遵循学校 5 维度评价标准框架。现行工作依赖纸质表格与人工汇总，存在数据分散、计算繁琐、维度否决规则难手算、结果难追溯等问题，需建设配套的信息化评分系统。

### 1.2 目标

- 全流程线上化：导入基础数据 → 分配评价任务 → 在线打分 → 自动计算 → 等级公示
- 严格落实李克特5级量表，屏蔽直接数字打分，保障评价客观性
- 自动执行学校 5 维度计分、去高去低、加权计算、维度否决
- 完整支持前置限定条件、S 级双通道申报、B/C/D 比例划定
- 支持学生评价记录免计入申请、持续改进跟踪单、申诉等配套流程
- 输出个性化成绩单与全院排名，符合学校归档要求

## 二、评价体系总体结构

### 2.1 学校 5 维度评价标准框架（核心）

所有评分表（听课/材料审查/同行说课/学生问卷）**严格按学校 5 维度框架**组织，每个维度有固定分值：

| 维度 | 分值 | S 级核心要求 |
|---|---|---|
| **1. 目标与思政** | **20** | 课程思政与专业知识有机融合；目标体现高阶能力；与毕业要求支撑关系清晰 |
| **2. 内容与方式** | **25** | 内容先进实用、项目/案例导引；学生中心方法；S级要求学生有效参与课堂≥90%；数字化工具有效运用 |
| **3. 考核与评价** | **20** | S级要求高阶能力考核占比≥50%；评分量规清晰；过程性记录完整；反馈及时有效 |
| **4. 效果与达成** | **20** | 学生达成证据明确；达成度计算合理；S级满意度≥90 |
| **5. 反思与改进** | **15** | 课程总结精准；改进计划具体可操作；上一轮问题有落实 |
| **合计** | **100** | |

### 2.2 三维评价权重

| 评价维度 | 权重 | 评分满分 | 实施主体 | 实施方式 |
|---|---|---|---|---|
| **上级评价**（委员评价） | **40%** | 100分 | 院级质量监控组委员 | 定向听课 + 教学材料审查 |
| **同行评价**（说课评价） | **30%** | 100分 | 全体参会教师 | 期末集中说课活动，现场打分 |
| **学生评价**（下级评价） | **30%** | 100分 | 修读该课程的学生 | 学校统一系统问卷（80分）+ 个别访谈（20分） |

> 综合评价分 S = 上级评价分×40% + 同行评价分×30% + 学生评价分×30%

### 2.3 上级评价内部结构

| 子项 | 占上级评价权重 | 评分表 |
|---|---|---|
| 定向听课 | 60% | 听课评分表（5维度×100分） |
| 教学材料审查 | 40% | 材料审查评分表（5维度×100分） |

> 上级评价分 = 听课平均分×60% + 材料审查平均分×40%

### 2.4 综合评价等级

| 等级 | 全称 | 比例 |
|---|---|---|
| **S** | 示范级 | ≤10%（中层干部不占） |
| **A** | 发展级 II | S+A 合计约40% |
| **B** | 发展级 I | 约30% |
| **C** | 关注级 | 约30% |
| **D** | 不合格 | 不限 |

### 2.5 维度否决规则（关键）

**若 5 个维度中任一维度得分率 < 70%，该教师评价等级不得为 A 及以上**（最高只能定为 B）。该规则在系统中自动检测并标记。

## 五、评分题目库（5维度框架）

### 5.1 统一题目结构

所有 4 套评分表均按学校 5 维度框架组织：

```
评分表
├── 维度1：目标与思政（满分20分）
│   └── 若干题目（题目分值之和=20分）
├── 维度2：内容与方式（满分25分）
│   └── 若干题目（题目分值之和=25分）
├── 维度3：考核与评价（满分20分）
│   └── 若干题目（题目分值之和=20分）
├── 维度4：效果与达成（满分20分）
│   └── 若干题目（题目分值之和=20分）
└── 维度5：反思与改进（满分15分）
    └── 若干题目（题目分值之和=15分）
```

### 5.2 李克特量表（统一）

| 显示选项 | 系统映射分值 |
|---|---|
| 完全符合 / 表现突出 | **5分（100%）** |
| 比较符合 / 表现较好 | **4分（80%）** |
| 基本符合 / 一般达成 | **3分（60%）** |
| 不太符合 / 存在明显不足 | **2分（40%）** |
| 完全不符合 / 严重不足 | **1分（20%）** |

每题最终得分 = 题目分值 × (李克特分数 ÷ 5)

> 评分界面**只显示文字，不显示分值与百分比**，防止锚定效应。

## 六、评分计算规则

### 6.1 单次评分计算

```
某评分人对某教师的某次评分：
  对每个维度 D：
    维度得分(D) = Σ(题目得分_in_D)
    维度满分(D) = Σ(题目分值_in_D)
    维度得分率(D) = 维度得分(D) ÷ 维度满分(D)
  总分 = Σ(所有维度得分) ≤ 100
```

### 6.2 上级评价分

```
听课平均分 = avg(委员每次听课总分)
材料审查平均分 = avg(委员每次审查总分)
上级评价分 = 听课平均分 × 60% + 材料审查平均分 × 40%
```

各维度得分率合并：取所有委员评分在该维度上的算术平均。

### 6.3 同行评价分

```
同行评价分 = (Σ有效打分 - max - min) ÷ (有效打分人数 - 2)   # 当人数≥3
同行评价分 = avg(所有打分)                                  # 当人数<3
```

### 6.4 学生评价分

```
学生问卷平均分 = avg(所有"已计入"的学生问卷100分制总分)
访谈平均分 = avg(各访谈委员20分制评分)
访谈归一化 = 访谈平均分 ÷ 20 × 100
学生评价分 = 学生问卷平均分 × 80% + 访谈归一化 × 20%
```

> "已计入"是指排除已通过"学生评价记录免计入申请"的记录。

### 6.5 综合评价分

```
综合评价分 S = 上级评价分 × 40% + 同行评价分 × 30% + 学生评价分 × 30%
```

### 6.6 维度得分率合并（用于否决判定）

每位教师每个维度的"加权综合得分率"按三维权重合并：

```
维度D加权得分率 = 上级维度D得分率 × 40% + 同行维度D得分率 × 30% + 学生维度D得分率 × 30%
```

**若任一维度的加权得分率 < 70%，触发"不得评A及以上"标记**。

### 6.7 数据完整性要求

| 维度 | 最低要求 | 不满足时的处理 |
|---|---|---|
| 听课 | 全学年≥2次 | 标记"数据不完整"，管理员决定 |
| 材料审查 | ≥1次审查 | 同上 |
| 同行评价 | ≥3人打分方可去极值 | <3人则取算术平均 |
| 学生问卷 | ≥10人提交 | 红色警告，管理员决定 |
| 个别访谈 | ≥2名委员 | 标记警告 |

## 七、等级认定规则

### 7.1 前置限定条件（自动标记，优先于综合分）

| 触发条件 | 处理结果 |
|---|---|
| 严重师德师风问题（经核实） | **直接 D 级** |
| 考核材料伪造（经核实） | **直接 D 级** |
| 学年学生问卷平均分 < 90 | **不得评 S 级** |
| 学年存在责任事故 | **不得评 B 级及以上** |
| 学年存在一般教学差错 | **不得评 A 级及以上** |
| **任一维度加权得分率 < 70%** | **不得评 A 级及以上** |

### 7.2 S 级评定（双通道申报）

- **通道1**：各系推评（系部根据综合分推荐）
- **通道2**：教师本人申报（提交示范案例材料）
- 院审定委员会初评 → 教务部组织校级评审 → 最终认定

系统支持：申报入口、材料上传、流程追踪、最终结果同步。

### 7.3 B/C/D 级自动划定

1. 排除已认定 S 级教师
2. 应用所有前置限定标记
3. 剩余教师按综合分排序
4. 按比例：A 级填充至 S+A=40%；B≈30%；C≈30%；D 不限
5. 边界情况由评价组商议
6. 中层管理职务教师不占 S/A 比例配额

- 系统校验：不得评自己课程；单一委员听课≤2次/同教师/年

### 8.3 上级评价打分

- 委员"待办任务"列表
- 5维度李克特评分表（按题目分值动态布局）
- 文字反馈（选填）
- 提交后锁定，仅管理员可解锁

### 8.4 同行说课现场打分

- 管理员创建场次、生成二维码
- 教师扫码进入打分页（移动端友好）
- WebSocket 实时同步
- 大屏实时显示：当前说课教师、已提交人数、实时去极值平均分、5维度得分分布
- 不允许为自己打分
- 提交锁定

### 8.5 学生问卷评价

- 时间窗口控制
- 学号登录，自动展示所修课程
- 匿名提交
- 5维度李克特问卷
- 提交锁定

### 8.6 个别访谈评分

- 管理员配置访谈信息
- 委员提交访谈分（20分制，3维度）
  - 能力培养感知（8分）
  - 教学方法接受度（6分）
  - 考核认可度（6分）
- 至少2名委员独立打分取平均

### 8.7 学生评价记录免计入申请（新增）

**流程**：
```
教师发起申请（填写：学生学号、班级、申请理由）
    ↓
系部主任审核（同意/驳回，附意见）
    ↓
学院秘书组审核（同意/驳回，附意见）
    ↓
学校教学质量保障部最终审核（同意/驳回）
    ↓
通过后：该学生评价记录标记为"免计入"，不参与计算
```

**功能要求**：
- 任课教师在评价窗口期内可提交申请
- 三级审核流程可见、可追溯
- 审核结论影响后续分数计算
- 申请记录归档保存

### 8.8 分数计算与维度否决

- 触发计算（管理员手动 或 数据齐全时自动）
- 计算流程：
  1. 各维度分别计算单评分人的得分率
  2. 跨评分人合并维度得分率
  3. 跨三维（上级/同行/学生）加权合并
  4. 综合分计算
  5. 维度否决判定（任一维度<70%）
  6. 前置限定标记应用
  7. 排序与等级划定

### 8.9 持续改进跟踪单管理（新增）

**适用对象**：C 级（关注级）教师

**字段**：
- 教师及课程名称
- 对应学年学期
- 所属教学单位
- 评价结果
- 存在的主要问题（清单）
- 整改举措 / 预计结果 / 预计验收时间 / 验收结果（多条记录）
- 教学单位审批意见
- 相关职能部门审批意见

**流程**：
```
学院与C级教师共同制定跟踪单
    ↓
学院秘书组备案 → 报送教学质量保障部备案
    ↓
系部主任跟踪（每学期≥1次专项听课）
    ↓
下一周期结束后专项评估
    ↓
评估结果计入下一周期综合评价
```

**功能要求**：
- 跟踪单模板与编辑
- 进度跟踪与提醒
- 验收结果记录
- 归档与查询（保存≥5年）
- 校院两级评估接口

### 8.10 中层管理职务教师评价（新增）

由学校层面单独评价，本院承担数据提供与配合职责：

- 系统标记"中层管理职务"教师
- 自动统计本院 S 级教师占比、增量数据
- 该类教师评价仅显示 S/A/D 三级
- 不参与本院 S/A 比例分配

### 8.11 申诉处理

- 教师在公示后3工作日内提交院内申诉
- 学院秘书组3工作日内复核答复
- 不服可向学校教学质量保障部提交校级复核（3工作日内）
- 申诉记录归档

### 8.12 结果展示与报表

**管理员仪表盘**：
- 各维度评价完成进度
- 数据异常预警
- 维度得分率热图（识别普遍薄弱环节）
- 全院综合评价分排名与等级分布
- 前置限定与维度否决触发情况

**教师个人成绩单**：
- 上级评价分（含听课、材料子项）
- 同行评价分
- 学生评价分（含问卷、访谈子项）
- **5 维度得分明细与雷达图**
- 综合评价分 S
- 维度否决标记（如有）
- 建议等级 / 最终等级

**报表导出**：
- 全院教师综合评价排名（Excel/PDF）
- 各维度明细数据
- 评价工具信度与效度分析报告
- 学校上报数据包（按学校归档要求）

## 附：工作流程对齐补充（2026-06-06，开发中确认）

实施过程中与项目方确认的执行细节，对原设计的澄清/细化（不改变三维权重与计分规则）：

1. **单点登录 + 角色化界面**：全体用户（管理员/委员/教师/学生）使用同一登录入口（账号+密码），登录后系统按其角色在**左侧导航**显示不同的可用功能，不设多个独立登录端。一人可兼多角色，导航显示其全部权限入口。

2. **课程来源改为教师自助填报**：取代人工批量导入，教师登录后在「参评课程填报」填报本学年**唯一一门**参评课程（含授课班级、是否课程负责人、是否教改课程）。管理员批量导入保留为后备。

3. **学生评教的师生匹配**：学生看到"本学期给自己上课的老师"，通过 **教师填报课程的`授课班级` ∩ 学生名单的`班级`** 匹配得出。学生仅就匹配到的教师（其参评课程）进行匿名评价。

4. **院级质量监控组名单采用导入方式（§4.4）**：单独导入一张名单 = 委员身份（质量委员/院长/系主任）+ 负责听课/材料审查的课程，系统据此 ① 授予对应角色 ② 批量生成听课/材料任务（复用任务分配引擎，仍受"≤2次/不评自己"校验）；管理员界面可个别调整。

5. **维持原设计的部分**：材料审查仍由委员**线上 5 维度打分**（保住维度否决所需的维度粒度）；同行评价仍走**现场说课同步打分**；不引入"中层干部给全员打分"；中层干部仍按原设计为"被单独评价、不占 S/A 比例"。

6. **成绩审核与发布采用「会签制」（严谨治理）**：
   - **审核委员会**：被标记为"审核委员会成员"的人员（院长、书记、中层干部）。教师导入表新增一列 `是否审核委员会成员`，谁标记谁参与会签（不写死人数）。
   - **发布**：系统算出建议等级后进入"待审核"。发布须委员会**全部成员逐一点"同意"（全票会签）**，全部同意 → 状态 PUBLISHED、记 publishedAt；**任一人驳回 → 本轮作废，需重新发起**。
   - **修改成绩 / 基础数据**：**禁止任何人单独修改**；须发起"变更申请"（写明变更内容与理由），同样经委员会**全部同意**后才生效。系统不提供"直接改等级"入口。
   - **留痕**：每位成员的同意/驳回 + 时间 + 意见，记入**独立的「审签记录」**（区别于普通操作日志），永久保存、可导出备查。
   - 数据模型补充：`User.isApprover`（审核委员会成员）、新表 `ApprovalRequest`（类型=发布/修改、变更内容、发起人、各成员会签结果、整体状态）。
   - **教师成绩可见性门禁**：仅当结果状态为 PUBLISHED 时教师才可见本人成绩单；发布前显示"成绩评定中"。
   - **成绩单含强弱诊断**：在三维子项+总分+5 维度雷达基础上，按维度得分率规则化生成文字诊断（最强/最弱维度、<70% 否决预警、对应改进建议）。

7. **课程数据改为「课表导入 + 教师选课」（取代手工填报/手工课程导入，后者下线）**：
   - **导入源 = 学校课表**：管理员分学期上传课表 xlsx（每次选课程类型：理论课文件→THEORY、实践课文件→PRACTICE），系统解析：`课程`列方括号取课程代码、括号外取名称；`教师`列方括号取**拼音账号**关联教师；`上课班级构成`按逗号拆班级（空行忽略）；`上课班级`编号忽略。
   - **去重落库**：按 `(课程代码 + 教师 + 学年)` 合并为一门开课，多行多班级取**并集**；系统生成开课 id，`Course.courseCode` 不再全局唯一（改为 `@@unique([courseCode, teacherId, academicYear])`）。重复导入按此键 **upsert 更新**。
   - **学年**：全部归入同一评价学年（导入时管理员填，默认 `2025-2026`）；学期信息忽略。
   - **教师账号**：`loginAccount = 拼音账号`（与课表一致）。课表中有、教师名单中无的教师 → **自动新建**（账号=拼音、姓名取括号外、初始密码 123456、强制改密、TEACHER 角色）。导入后给匹配/新建/失败行报告。
   - **教师端**：原"参评课程填报"下线，改为 **"从我的课程里选一门参评"**（课程来自课表，教师只选不填，可改课程类型/勾课程负责人）。学生按 `班级∈该课上课班级构成` 自动关联匿名评教。
