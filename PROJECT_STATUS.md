## 📊 方案核心要点速览（评审参考）

### 1. 评价标准框架（学校 5 维度，强制）

| 维度 | 分值 | 关键 S 级标准 |
| --- | --- | --- |
| 1. 目标与思政 | **20** | 思政融合、高阶能力、毕业要求支撑 |
| 2. 内容与方式 | **25** | 项目导引、学生参与≥90%、数字工具 |
| 3. 考核与评价 | **20** | 高阶考核≥50%、量规清晰 |
| 4. 效果与达成 | **20** | 达成度≥0.70、满意度≥90 |
| 5. 反思与改进 | **15** | 精准分析、改进可操作 |
| **合计** | **100** |   |

### 2. 三维评价权重

| 评价主体 | 权重 | 子项 |
| --- | --- | --- |
| 上级评价 | 40% | 听课 60% + 材料 40% |
| 同行评价 | 30% | 说课现场打分（去极值） |
| 学生评价 | 30% | 问卷 80% + 访谈 20% |

### 3. 等级体系

| 等级 | 全称 | 比例 |
| --- | --- | --- |
| S | 示范级 | ≤10% |
| A | 发展级 II | S+A≈40% |
| B | 发展级 I | ≈30% |
| C | 关注级 | ≈30% |
| D | 不合格 | 不限 |

### 4. 关键否决规则（务必评审确认）

> **任一维度得分率 < 70% → 不得评 A 及以上**（最高 B 级）

### 5. 前置限定（直接评定）

| 情形 | 结果 |
| --- | --- |
| 严重师德问题 | 直接 D |
| 考核材料伪造 | 直接 D |
| 学生评分 < 90 | 不得 S |
| 责任事故 | 不得 B 及以上 |
| 教学差错 | 不得 A 及以上 |

## ✅ 评审建议关注点（Checklist）

请评审委员对照以下要点确认方案：

### 合规性

- 5 维度框架（20+25+20+20+15）是否与学校《评价办法》附件 2 一致
- 任一维度<70% 不得 A 及以上规则是否在方案中明示
- 课程覆盖比例 70%/100% 是否符合学校要求
- 等级中文名（A=发展级II、B=发展级I）是否准确
- 申诉期限 3 工作日是否一致

### 公平性

- 上级 40% / 同行 30% / 学生 30% 权重是否合理
- 听课 60% + 材料 40% 的上级评价内部权重是否合适
- 同行评价去极值是否在 3 人以上才生效
- 中层干部不占 S/A 比例是否会引起争议
- 教改加难度课程的阈值调整流程是否清晰

### 可操作性

- 院级质量监控组成员名单是否最新（已含 18 人）
- 时间节点是否与本学年校历对齐
- 学生评价免计入三级审核流程的责任部门是否明确
- 持续改进跟踪单的归口部门与备案要求是否清晰

### 反馈与申诉

- 个性化成绩单内容是否够支持教师自我诊断
- 院内 3 工作日 + 校级 3 工作日的申诉时序是否便于操作
- C 级跟踪帮扶的"结对机制"是否可执行

### 系统设计（如需评审技术细节）

- 5 维度评分提交时缓存得分率，是否便于后续否决判定
- 学生匿名双表设计是否满足合规要求
- 现场说课的 WebSocket 方案是否满足并发要求
- 题目按课程类型版本化是否便于后续维护
